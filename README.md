## 项目概述

TopView2026（赛博聚石盆）是一个基于 Java Web 技术栈开发的视频社交平台，支持视频发布、Feed 流推送、优惠券抢购等核心功能。

### 考核信息

- **一轮考核**：基础视频平台功能（已完成）
- **二轮考核**：Feed 流推送 + 优惠券抢购（进行中）
- **开始时间**：2026-04-23
- **截止时间**：2026-05-11
- **答辩时间**：2026-05-12

### 核心特性

✅ **手写 MVC 框架**：不使用 SpringBoot/SpringMVC/MyBatis  
✅ **前后端分离**：后端提供 RESTful API，前端使用 Vue/React  
✅ **Redis 缓存**：用户行为数据缓存、Feed 流加速  
✅ **OSS 云存储**：阿里云 OSS 视频文件存储  
✅ **RBAC 权限控制**：基于角色的访问控制  
✅ **声明式事务**：自定义注解实现事务管理  
✅ **连接池优化**：自定义数据库连接池

##### 技术实现要点

**Redis 数据结构设计**
- `feed:push:{userId}`: ZSet 存储推送给用户的视频ID列表，score 为发布时间戳
- `video:hotness`: 缓存视频热度分数（可选）

---

**热度排序算法（牛顿冷却定律）**
- **公式**: `热度分数 = 互动得分 / (经过小时数 + 2) ^ 重力因子`
- **互动得分**: 播放量×1 + 点赞数×5 + 评论数×10
- **重力因子**: 1.8（控制衰减速度）
- **应用场景**: Feed流视频排序、评论热度排序
- **优势**:
    - 新内容有机会快速上升（挖坟机制）
    - 老内容自然下沉，避免长期霸榜
    - 平衡时效性与质量

## 常见问题

### Q1: 启动时提示 "ApplicationContext not initialized"

**原因**: AppStartupListener 未正确加载  
**解决**: 检查 `web.xml` 中是否配置了 Listener

### Q2: Redis 连接失败

**原因**: Redis 未启动或配置错误  
**解决**: 

### Q3: OSS 上传返回 403

**原因**: AccessKey 权限不足或 Bucket 配置错误  
**解决**: 检查 `db.properties` 中的 OSS 配置

### Q4: 视频缓存下载失败（403）

**原因**: OSS Bucket 为私有权限  
**解决**: 
1. 使用签名 URL（已实现）
2. 或将 Bucket 改为公共读

### Q5: 事务不回滚

**原因**: 异常被捕获未抛出  
**解决**: 确保异常向上抛出，或使用 `@Transactional` 注解

   - 方法/变量：小驼峰（camelCase）
