## 功能概述

本模块实现短视频场景下的优惠券抢购功能，支持博主发布视频时选择不同优惠策略，满足不同运营场景需求。

**核心技术栈：**
- ✅ Redis + MySQL 事务 + 乐观锁
- ✅ 保证高并发下库存安全、不超卖、数据一致
- ✅ 支持多种抢购模式（普通抢购、分批次、预约抽签、观看解锁）

## 必做方案：普通限量抢购

### 业务场景

- 🎯 视频挂载固定库存优惠券
- ⏰ 活动时间内有效，先到先得
- 🚫 库存抢完后，即使活动未结束也无法继续抢购

### 核心流程
1. **用户点击抢购**
2. **校验阶段**
   - 登录状态校验
   - 活动时间校验
   - 用户是否已领取校验
3. **Redis 原子扣减库存**（DECR）
4. **数据库事务**
   - 扣减库存
   - 生成用户优惠券记录
5. **同步返回抢购结果**

### 技术亮点

- ✅ **纯 Redis 原子操作**保证并发安全
- ✅ **MySQL 事务**保证数据一致性
- ✅ **实现简单、稳定**，适合基础秒杀场景

## 进阶选做方案

### 方案1：分批次动态放券

#### 业务场景

解决"早刷到能抢、晚刷到无库存"的问题，活动期间分多轮放出库存。

#### 实现思路

1. **总库存拆分**为多批次（如 3 批）
2. **配置每批次**发放数量、发放时间
3. **定时任务/后台触发**，动态往 Redis 补充库存
4. **抢购逻辑**与普通抢购完全复用

#### 加分点

- ✨ 实现抢购期间动态调整库存
- ✨ 晚刷到视频的用户仍有机会参与

### 方案2：预约抽签券

#### 业务场景

不做先到先得，活动期间只允许预约，结束后统一抽签发放，保证公平性。

#### 实现思路

1. **活动期间**：用户点击"预约抢券"，仅记录预约关系
2. **不扣库存**、不生成券码
3. **活动结束后**：
   - 从所有预约用户中随机抽取
   - 抽中才扣库存、生成优惠券
4. **结果通知**：通过异步通知推送给用户

#### 适用场景

- 🎁 福利券
- 🏆 纪念券
- 🎨 品牌活动等强调公平的场景

### 方案3：观看时长解锁券

#### 业务场景

用户必须观看视频达到指定时长，才能解锁抢购资格，提高视频完播率。

#### 实现思路

1. **前端记录**用户观看时长
2. **达到要求后**请求后端解锁资格
3. **后端校验**通过后，才允许进入抢购流程
4. **后续逻辑**复用普通抢购（库存扣减、事务）

#### 亮点

- ✨ **深度贴合短视频业务场景**
- ✨ **防止恶意秒抢**、提升内容播放数据

## 技术架构

### 核心技术

| 技术 | 用途 | 说明 |
|------|------|------|
| **Redis** | 库存管理 | DECR 原子操作，防止超卖 |
| **MySQL** | 持久化存储 | 事务保证数据一致性 |
| **乐观锁** | 并发控制 | UPDATE ... WHERE stock > 0 |
| **Lua 脚本** | 原子性保证 | 库存扣减 + 资格校验 |

### 防止超卖方案

#### 方案 1：Redis DECR（推荐）

#### 方案 2：MySQL 乐观锁

#### 方案 3：Lua 脚本（最优）
